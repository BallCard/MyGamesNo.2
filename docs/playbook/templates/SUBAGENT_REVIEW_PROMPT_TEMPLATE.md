# Subagent 审阅提示词模板

## 1. 使用场景

适用于：

- 请 subagent 审玩法和手感
- 请 subagent 审 UI / UX
- 请 subagent 审 debug 方向
- 请 subagent 审文档或测试边界

不适用于：

- 让 subagent 直接接管主线架构
- 在没有上下文时让 subagent 直接大改代码

## 3. 希望 subagent 输出的格式

请按下面结构输出：

1. 你认为当前问题更像是：设计问题 / 数值问题 / 状态机问题 / 输入链问题 / UI 映射问题
2. 你最认同的根因排序（最多 3 条）
3. 你建议优先尝试的修改顺序
4. 你明确不建议做的事
5. 如果要继续迭代，你建议主 agent 下一轮只改哪一个变量

## 4. 主 agent 使用规则

- subagent 的输出是建议，不是最终裁决
- 主 agent 负责最终技术收口
- 主 agent 只采纳与当前架构兼容的建议
- 若建议会扩大耦合或打破既定边界，应拒绝采纳

## 5. 推荐提问方式

### A. 审玩法手感

请从“轻挫败、爽感反馈、可传播性”三个维度审当前版本，并指出最值得优先调整的一项。

### B. 审 debug 方向

请判断当前 bug 更可能属于哪一层，并给出最小验证顺序，不要直接给大改方案。

### C. 审 UI

请只评估视觉层级、信息密度、按钮可读性和移动端操作舒适度，不要改动核心玩法规则。
